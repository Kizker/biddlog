Panduan Deployment Next.js Docker ke VPS Hostinger

Projek: Trading Simulation
Versi: 2.0 (Berdasarkan Konfigurasi Docker Next.js + PostgreSQL)
Domain: simvest.site
IP VPS: 72.62.127.119

Panduan ini disusun untuk memberikan instruksi step-by-step dari awal hingga akhir dalam mendistribusikan projek Trading Simulation Anda menggunakan infrastruktur Docker pada VPS Hostinger.

Fasa 1: Persediaan di VPS Hostinger

1. SSH Login ke VPS

Buka Terminal/PowerShell/PuTTY, lalu login ke server Hostinger Anda:

ssh root@72.62.127.119

(Masukkan password VPS Anda saat diminta)

2. Kemaskini Peladen & Pasang Docker

Setelah berhasil login, jalankan perintah (salin-tempel) berikut ini sekaligus untuk memasang mesin Docker di Ubuntu Anda:

sudo apt update && sudo apt upgrade -y

# Pasang sijil yang diperlukan
sudo apt install -y ca-certificates curl gnupg

# Tambah GPG key rasmi Docker
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Tetapkan repositori Docker
echo \
  "deb [arch="$(dpkg --print-architecture)" signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  "$(. /etc/os-release && echo "$VERSION_CODENAME")" stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Pasang Docker dan Docker Compose
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin -y

# Beri kebenaran agar Docker boleh dijalankan tanpa 'sudo'
sudo usermod -aG docker $USER

3. Klon Projek dari GitHub

Siapkan direktori server Anda dan tarik repositori GitHub projek Trading Simulation:

mkdir -p /var/www/fastuser/data/www/simvest.site
cd /var/www/fastuser/data/www/simvest.site
git clone https://github.com/FauzanAmirudin/trading-simulation.git .

(Catatan: Penggunaan titik . di akhir perintah clone berfungsi agar file langsung diekstrak di dalam folder tersebut).

Fasa 2: Tetapan Environment (.env)

Kita harus membuat file konfigurasi .env dengan kredensial untuk database PostgreSQL dan aplikasi Next.js.

  1. Buat file baru bernama .env:
nano .env
  1. Salin teks berikut seluruhnya dan tempel (paste) ke dalam jendela Nano Anda:
# Database Config untuk PostgreSQL Container
POSTGRES_USER=simvest_site
POSTGRES_PASSWORD=@Dea18022003
POSTGRES_DB=simvest_site

# URL Koneksi Database untuk Next.js (Drizzle)
# Format: postgresql://[USER]:[PASSWORD]@postgres:5432/[DB]
# Pastikan ini sesuai dengan konfigurasi POSTGRES_ di atas.
# 'postgres' adalah nama service database di docker-compose.yml
DATABASE_URL=postgresql://simvest_site:@Dea18022003@postgres:5432/simvest_site
  1. Simpan dengan menekan tombol Ctrl + X, lalu ketik Y, dan tekan Enter.

Fasa 3: Eksekusi dan Konfigurasi Kontena

Kini semua pengaturan telah siap, mari kita jalankan server.

1. Bina dan Jalankan Docker

Proses ini akan mengunduh PostgreSQL, melakukan build aplikasi Next.js, dan akan memakan waktu beberapa menit. Saat dijalankan, container juga akan secara otomatis mengaplikasikan skema database (Push DB schema) menggunakan drizzle-kit berdasarkan konfigurasi di Dockerfile.

docker compose up -d --build

2. Memasukkan Data Awal (Seeding) - Opsional

Jika Anda memerlukan data dummy atau seed, Anda bisa menjalankannya setelah container berjalan. Karena image menggunakan tsx untuk eksekusi, Anda dapat melakukan docker exec untuk menjalankan script seed bawaan dari package.json.

docker exec -it trading_app npm run db:seed

Fasa 4: Konfigurasi Domain (simvest.site) dengan Nginx

Aplikasi saat ini berjalan di port 3000 pada VPS Anda. Agar bisa diakses secara rapi menggunakan domain simvest.site, Anda perlu melakukan setup Reverse Proxy menggunakan Nginx.

1. Install Nginx

sudo apt install nginx -y

2. Buat File Konfigurasi Nginx untuk Domain

sudo nano /etc/nginx/sites-available/simvest.site

3. Masukkan Konfigurasi Berikut:

Salin dan paste blok di bawah ini. Pastikan Anda telah mengarahkan A Record domain simvest.site ke IP 72.62.127.119 di DNS management Anda.

server {
    listen 80;
    server_name simvest.site www.simvest.site;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Simpan dengan Ctrl + X, ketik Y, lalu Enter.

4. Aktifkan Konfigurasi dan Restart Nginx

sudo ln -s /etc/nginx/sites-available/simvest.site /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

5. Selesai!

Aplikasi Trading Simulation Anda kini dapat diakses melalui http://simvest.site.

6. Mengaktifkan HTTPS (SSL Let's Encrypt) di FASTPANEL

FASTPANEL secara default menggunakan Nginx versi kustom. Karena adanya bug terkait kunci ECDSA pada FASTPANEL, Anda wajib membuat kunci bertipe RSA agar tidak ditolak dengan pesan error "Sertifikat tidak cocok dengan kunci privat".

# Hentikan Nginx bawaan
sudo systemctl stop nginx

# Instal Certbot & Generate Sertifikat (Wajib tipe RSA)
sudo apt install certbot -y
sudo certbot certonly --standalone -d simvest.site -d www.simvest.site --key-type rsa --force-renewal

# Jalankan kembali Nginx
sudo systemctl start nginx

Setelah sertifikat terbentuk, masukkan sertifikat tersebut ke panel FASTPANEL (Menu Sertifikat SSL -> Yang ada). Anda dapat melihat teks kunci privat dan sertifikatnya menggunakan perintah berikut:

# Lihat Kunci Privat
cat /etc/letsencrypt/live/simvest.site/privkey.pem

# Lihat Sertifikat Utama & Rantai CA
cat /etc/letsencrypt/live/simvest.site/fullchain.pem

Catatan: Pisahkan blok sertifikat pertama ke kotak Sertifikat, dan sisa blok berikutnya ke kotak Rantai. Jangan lupa aktifkan Pengalihan HTTPS di menu Pengaturan situs.
