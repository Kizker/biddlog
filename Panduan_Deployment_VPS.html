Konfigurasi DNS Domain

Atur A Record di Hostinger DNS

  1. Login ke hpanel.hostinger.com
  2. Pilih domain biddlog.site → klik Domain → DNS/Nameserver
  3. Pastikan hanya ada SATU A record:
TipeNamaValueTTL
A@72.62.127.119300
⚠️ PENTING: Hapus A record lama yang mengarah ke IP selain 72.62.127.119 (misalnya 2.57.91.91 yang merupakan IP parkir bawaan Hostinger). Jika ada 2 A record, SSL akan GAGAL!

Setelah diubah, tunggu 5-15 menit agar DNS terpropagasi.

2

Buat Situs di FASTPANEL

Login & Buat Site FASTPANEL

  1. Buka browser → https://72.62.127.119:8888
  2. Login dengan user fastuser (password ada di dashboard Hostinger VPS)
  3. Klik Situs → Buat Situs → pilih Manual
  4. Pengikatan domain: masukkan biddlog.site
  5. Konfigurasi: Backend type → pilih Konten statis
  6. Klik Buat situs
💡 Kenapa Konten statis dulu? SSL Let's Encrypt hanya bisa diterbitkan ketika backend BUKAN proxy terbalik. Nanti setelah SSL berhasil, kita ubah ke Proxy terbalik.
⚠️ Jika situs nyangkut / gagal dibuat karena error DKIM "Debian-exim":
Jalankan perintah ini di SSH untuk membuat user email kosong yang dicari FASTPANEL:
mkdir -p /etc/exim4 && touch /etc/exim4/dkim_selector
groupadd Debian-exim && useradd -g Debian-exim Debian-exim
Setelah itu coba hapus/buat situs lagi.
3

Pasang SSL HTTPS (Let's Encrypt)

⚠️ Urutan kritis: SSL HARUS dipasang SEBELUM reverse proxy diaktifkan dan SEBELUM Docker dijalankan. Ini adalah pelajaran dari deployment sebelumnya.

Metode A: Via FASTPANEL (Paling Mudah) FASTPANEL

  1. Di FASTPANEL, klik situs biddlog.site
  2. Masuk ke menu Sertifikat SSL
  3. Klik Sertifikat Baru → pilih Let's Encrypt
  4. KUNCI SUKSES:
    • Kotak Nama Alternatif: KOSONGKAN TOTAL
    • Toggle Wajibkan validasi DNS: MATI / OFF
  5. Klik Simpan → tunggu beberapa detik
✅ Berhasil jika: Status sertifikat berubah hijau/aktif.
⚠️ Jika gagal: Pastikan A record DNS hanya mengarah ke 72.62.127.119. Jika DNS baru diubah, tunggu 15-30 menit lalu coba lagi.

Metode B: Certbot Standalone (Manual) SSH

Jika FASTPANEL gagal, Anda bisa menggunakan sertifikat manual. Wajib gunakan tipe RSA agar FASTPANEL bisa membacanya tanpa error "Sertifikat tidak cocok".

SSH — Certbot standalone (RSA)
# Install certbot
apt install -y certbot

# Matikan Nginx sementara
systemctl stop nginx

# Dapatkan sertifikat TIPE RSA
certbot certonly --standalone -d biddlog.site --key-type rsa --force-renewal

# Hidupkan Nginx kembali
systemctl start nginx

Setelah berhasil, masukkan sertifikat ke FASTPANEL (Sertifikat SSL → Sertifikat Baru → pilih "Yang ada"):

  1. Kunci Privat: jalankan cat /etc/letsencrypt/live/biddlog.site/privkey.pem di SSH, copy-paste isinya.
  2. Sertifikat: jalankan cat /etc/letsencrypt/live/biddlog.site/cert.pem di SSH, copy-paste isinya.
  3. Rantai: jalankan cat /etc/letsencrypt/live/biddlog.site/chain.pem di SSH, copy-paste isinya.
  4. Klik Simpan
💡 Tips: Dengan membaca cert.pem dan chain.pem terpisah (bukan fullchain.pem), Anda dijamin tidak akan salah potong baris!

Aktifkan Redirect HTTPS FASTPANEL

Install Docker di VPS

Login SSH & Install Docker Engine SSH

Terminal Laptop
ssh root@72.62.127.119

Jalankan instalasi Docker (copy-paste semua sekaligus):

SSH — Install Docker (sekali jalan)
# Update sistem
apt update && apt upgrade -y

# Install dependencies
apt install -y ca-certificates curl gnupg lsb-release git

# Tambah GPG key Docker
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
chmod a+r /etc/apt/keyrings/docker.gpg

# Tambah repository Docker
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker Engine + Docker Compose
apt update
apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Verifikasi
docker --version
docker compose version
✅ Berhasil jika muncul: Docker version 2x.x.x dan Docker Compose version v2.x.x
5

Clone & Deploy dengan Docker

Clone Repository & Jalankan Docker SSH

File Dockerfile, nginx.conf, .dockerignore, dan docker-compose.yml sudah ada di dalam repository. Tinggal clone dan jalankan:

SSH — Clone & deploy (sekali jalan)
# Masuk ke folder situs
cd /var/www/fastuser/data/www/biddlog.site

# Bersihkan file default FASTPANEL
rm -rf ./*

# Clone repository (semua file Docker sudah ada di dalamnya)
git clone https://github.com/Kizker/biddlog.git .

# Build dan jalankan container
docker compose up -d --build

# Cek status
docker compose ps

# Test akses lokal
curl -I http://127.0.0.1:8080
✅ Berhasil jika: docker compose ps menunjukkan container bidlog_dashboard berstatus Up, dan curl mengembalikan HTTP/1.1 200 OK.
⚠️ Jika error "container name already in use":
Jalankan docker rm -f bidlog_dashboard lalu ulangi docker compose up -d --build
6

Aktifkan Reverse Proxy

Arahkan FASTPANEL ke Docker FASTPANEL

  1. Login FASTPANEL → klik situs biddlog.site
  2. Masuk ke Pengaturan → klik ikon edit di bagian Backend
  3. Ubah Backend type dari Konten statis menjadi Proxy terbalik
  4. Klik tombol + Add
  5. Isi alamat upstream: http://127.0.0.1:8080
  6. Klik Simpan
💡 Apa yang terjadi: FASTPANEL akan meneruskan semua request dari https://biddlog.site ke container Docker yang berjalan di port 8080. SSL/HTTPS ditangani oleh FASTPANEL, container Docker hanya melayani HTTP biasa.
